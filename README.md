# loglint

`loglint` — это кастомный статический анализатор для Go, который проверяет корректность текстов лог-сообщений в коде.  
Линтер реализован на базе фреймворка `go/analysis` и может использоваться как:

- самостоятельный CLI-инструмент
- плагин для `golangci-lint`

Основная цель — автоматически находить некорректные лог-сообщения и поддерживать единый стиль логирования в проекте.

---

# Описание задания

Необходимо разработать линтер для Go, совместимый с golangci-lint,
который будет анализировать лог-записи в коде и проверять их
соответствие установленным правилам.

Линтер должен проверять следующие правила для лог-записей:

1. Лог-сообщения должны начинаться со строчной буквы
2. Лог-сообщения должны быть только на английском языке
3. Лог-сообщения не должны содержать спецсимволы или эмодзи
4. Лог-сообщения не должны содержать потенциально чувствительные
данные

Дополнительно требовалось:

- написать unit-тесты для анализатора
- реализовать запуск линтера как CLI-программы
- реализовать сборку плагина для `golangci-lint`
- настроить CI для автоматической проверки проекта

---

# Проверяемые правила

Линтер анализирует строку лог-сообщения и проверяет следующие правила:

### 1. Сообщение должно начинаться со строчной буквы

Неверно

```go
slog.Info("Starting server")
```

Верно

```go
slog.Info("starting server")
```

---

### 2. Сообщение должно быть на английском языке

Неверно

```go
slog.Error("ошибка подключения")
```

Верно

```go
slog.Error("connection error")
```

---

### 3. Запрещены специальные символы и эмодзи

Неверно

```go
slog.Warn("failed!")
```

Верно

```go
slog.Warn("operation failed")
```

---

### 4. Запрещено логировать чувствительные данные

Неверно

```go
password := "123"
slog.Info("user password: " + password)
```

Такие сообщения могут приводить к утечке конфиденциальной информации.

# Архитектура решения

Проект построен на базе стандартного фреймворка Go для статического анализа:

```
golang.org/x/tools/go/analysis
```

Основные компоненты решения:

### Analyzer

Анализатор:

* обходит AST исходного кода
* находит вызовы логирования
* извлекает строку сообщения
* применяет набор правил
* формирует диагностические сообщения

Для обхода AST используется `inspect`-анализатор.

---

### Unit-тесты

Тесты написаны с использованием пакета:

```
analysistest
```

Он позволяет:

* компилировать тестовые Go-файлы
* запускать анализатор
* проверять ожидаемые ошибки через комментарии `// want`.

Пример:

```go
slog.Info("Starting") // want "log message should start with a lowercase letter"
```

---

### CLI-инструмент

Для запуска линтера как обычной программы используется:

```
singlechecker
```

Пример запуска:

```
go build -o loglint ./cmd/loglint
./loglint ./...
```

Пример результата:

```
example.go:6:12: log message should start with a lowercase letter
example.go:7:13: log message should be in English
```

---

### Плагин для golangci-lint

Линтер может использоваться как кастомный плагин.

Для этого реализован wrapper-пакет:

```
cmd/plugin
```

Сборка плагина:

```
go build -buildmode=plugin -o loglint.so ./cmd/plugin
```

После этого плагин можно подключить в `.golangci.yml`.

# CI

Для проекта настроен CI через GitHub Actions.

Workflow выполняет:

1. установку Go
2. запуск unit-тестов
3. сборку плагина

Файл:

```
.github/workflows/build.yml
```

Основные шаги:

```
go test ./...
go build -buildmode=plugin -o loglint.so ./cmd/plugin
```

CI выполняется на Linux-runner, поскольку сборка Go-плагинов не поддерживается на Windows.
